## 一、快速开始

### 1.1 访问服务器

使用 SSH 工具（如 PuTTY、Xshell）登录到你的 VPS：

```bash
ssh root@你的服务器IP
```

### 1.2 进入博客目录

```bash
cd /var/www/hexo-blog
```

### 1.3 基本操作流程

```
编辑内容 → 重新生成 → 查看效果
```

**标准工作流程**：
1. 编辑文章或配置文件
2. 上传图片/视频（如果需要）
3. 运行生成命令：`npx hexo clean && npx hexo generate`
4. 刷新浏览器查看效果

## 三、创建和管理文章

### 3.1 创建新文章

#### 方法一：使用命令创建（推荐）

```bash
# 进入博客目录
cd /var/www/hexo-blog

# 创建新文章
npx hexo new "宝贝的第一次微笑"
```

执行后会在 `source/_posts/` 目录下生成 `宝贝的第一次微笑.md` 文件

#### 方法二：手动创建

```bash
cd /var/www/hexo-blog/source/_posts
nano my-story.md
```

**两种方法的区别**：
- 方法一：自动生成标准格式的 Front-matter
- 方法二：需要手动编写所有内容

### 3.2 文章结构详解

一篇完整的文章由两部分组成：

```markdown
---
# 这里是 Front-matter（文章元信息）
title: 文章标题
date: 2024-01-15 10:30:00
---

<!-- 这里是正文内容 -->
文章正文使用 Markdown 语法编写
```

### 3.3 Front-matter 字段详解

Front-matter 是文章开头 `---` 之间的配置区域，用于设置文章属性。

#### 基础字段

```yaml
---
title: 宝贝的第一次微笑          # 文章标题
date: 2024-01-15 10:30:00      # 发布日期和时间
---
```

#### 分类和标签

```yaml
---
# 单个分类
categories: 成长记录

# 多个分类（层级结构）
categories: 
  - 成长记录
  - 重要时刻

# 单个标签
tags: 微笑

# 多个标签
tags:
  - 微笑
  - 开心
  - 2024年1月
---
```

**分类 vs 标签的区别**：
- **分类（categories）**：有层级关系，一篇文章通常属于1-2个分类
  - 例如：成长记录 > 第一次
- **标签（tags）**：无层级，可以有多个，更灵活
  - 例如：微笑、开心、温馨、1岁

#### 封面和摘要

```yaml
---
cover: /img/cover1.jpg                    # 文章封面图
description: 今天宝贝第一次笑了         # 文章摘要（显示在首页）
---
```

**cover（封面图）**：
- 显示位置：首页文章列表、文章顶部
- 图片路径：必须以 `/img/` 开头
- 推荐尺寸：1200x630 或 16:9 比例

**description（摘要）**：
- 显示位置：首页文章卡片
- 长度建议：50-100 字
- 不设置时：自动截取正文前几行

#### 高级字段

```yaml
---
top: true              # 置顶文章（显示在最前面）
comments: false        # 关闭评论功能
hide: false           # 隐藏文章（不在列表显示）
password: 123456      # 文章密码保护
abbrlink: baby-smile  # 自定义 URL 链接
---
```

#### 完整示例

```yaml
---
title: 宝贝的第一次微笑
date: 2024-01-15 10:30:00
updated: 2024-01-16 08:00:00
categories:
  - 成长记录
  - 重要时刻
tags:
  - 微笑
  - 开心
  - 2024年1月
cover: /img/2024-01-smile-cover.jpg
description: 今天是个特别的日子，颜颜宝贝对着爸爸妈妈露出了人生中的第一个笑容！那甜甜的笑容，温暖了整个家
top: false
comments: true
---
```

### 3.4 Markdown 语法详解

#### 标题

```markdown
# 一级标题（最大）
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题（最小）
```

**使用建议**：
- 文章标题用 Front-matter 的 `title` 设置
- 正文从二级标题 `##` 开始
- 保持标题层级清晰

#### 文本样式

```markdown

*这是斜体文字*
***这是粗斜体***
~~这是删除线~~
```

- **这是粗体文字**

## 四、路径说明详解（重要）

### 4.1 路径基础知识

在使用博客时，你会遇到两种路径：**服务器实际路径**和**引用路径**。理解它们的区别非常重要。

#### 4.1.1 服务器实际路径（上传文件时使用）

这是文件在服务器上的真实存储位置，用于上传文件、编辑文件等操作。

```
完整路径结构：
/var/www/hexo-blog/source/资源目录/文件名
└─────┬────────┘└──┬──┘└─┬─┘
      │            │     └─ 资源目录：img/、videos/、music/
      │            └─ 源文件目录（固定）
      └─ 博客根目录（固定）
```

**常用实际路径一览表：**

| 资源类型 | 服务器实际路径（完整） | 说明 |
|---------|---------------------|------|
| 图片 | `/var/www/hexo-blog/source/img/文件名.jpg` | 所有图片统一存放 |
| 视频 | `/var/www/hexo-blog/source/videos/文件名.mp4` | 视频文件存放 |
| 音乐 | `/var/www/hexo-blog/source/music/文件名.mp3` | 音乐文件存放 |
| 文章 | `/var/www/hexo-blog/source/_posts/文章名.md` | 文章 Markdown 文件 |

**示例：上传图片到服务器**
```bash
# 使用 SCP 上传图片
scp /本地/照片.jpg root@服务器IP:/var/www/hexo-blog/source/img/

# 上传后，文件实际位置是：
# /var/www/hexo-blog/source/img/照片.jpg
```

#### 4.1.2 引用路径（在文章中使用）

这是在文章、配置文件中引用资源时使用的路径，以 `/` 开头，**省略了** `/var/www/hexo-blog/source` 部分。

```
引用路径格式：
/资源目录/文件名
└───┬───┘└─┬─┘
    │      └─ 文件名（含扩展名）
    └─ 资源目录：img、videos、music（注意：没有最后的 /）
```

**引用路径一览表：**

| 资源类型 | 引用路径格式 | 实际对应的服务器路径 |
|---------|------------|-------------------|
| 图片 | `/img/文件名.jpg` | `/var/www/hexo-blog/source/img/文件名.jpg` |
| 视频 | `/videos/文件名.mp4` | `/var/www/hexo-blog/source/videos/文件名.mp4` |
| 音乐 | `/music/文件名.mp3` | `/var/www/hexo-blog/source/music/文件名.mp3` |

**⚠️ 重要规则：**
1. 引用路径**必须**以 `/` 开头（绝对路径）
2. 目录名固定：`img`、`videos`、`music`（全小写，videos 有 s）
3. 文件名**区分大小写**：`Photo.jpg` 和 `photo.jpg` 是不同的文件

### 4.2 路径对应关系图

```
服务器实际路径 ←→ 引用路径的转换

┌─────────────────────────────────────────────────────────────────┐
│ 服务器实际路径（上传位置）                                          │
├─────────────────────────────────────────────────────────────────┤
│ /var/www/hexo-blog/source/img/baby-smile.jpg                    │
│                            └──────┬─────────┘                    │
│                                   │                              │
│                            去掉 source 前缀                       │
│                                   │                              │
│                                   ▼                              │
│                        /img/baby-smile.jpg                       │
│                        └─────┬──────────┘                        │
├──────────────────────────────┴──────────────────────────────────┤
│ 引用路径（文章中使用）                                             │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 常见路径错误

#### ❌ 错误写法 vs ✅ 正确写法

**错误 1：忘记开头的斜杠**
```markdown
❌ 错误：cover: img/photo.jpg
✅ 正确：cover: /img/photo.jpg
```

**错误 2：使用了服务器实际路径**
```markdown
❌ 错误：![图片](/var/www/hexo-blog/source/img/photo.jpg)
✅ 正确：![图片](/img/photo.jpg)
```

**错误 3：使用了相对路径**
```markdown
❌ 错误：![图片](../img/photo.jpg)
❌ 错误：![图片](./img/photo.jpg)
✅ 正确：![图片](/img/photo.jpg)
```

**错误 4：目录名拼写错误**
```markdown
❌ 错误：/image/photo.jpg      (应该是 img 不是 image)
❌ 错误：/video/test.mp4       (应该是 videos 有 s)
❌ 错误：/IMG/photo.jpg        (应该是小写 img)
✅ 正确：/img/photo.jpg
✅ 正确：/videos/test.mp4
```

**错误 5：文件名大小写错误**
```markdown
# 如果服务器上的文件是：Photo.jpg（大写 P）

❌ 错误：/img/photo.jpg    (小写 p，Linux 系统区分大小写)
✅ 正确：/img/Photo.jpg    (大写 P，与实际文件名一致)
```

**错误 6：文件扩展名错误**
```markdown
# 如果实际文件是：photo.jpg

❌ 错误：/img/photo.png    (扩展名不对)
❌ 错误：/img/photo        (缺少扩展名)
✅ 正确：/img/photo.jpg
```

### 4.5 路径验证方法

#### 方法 1：在服务器上检查文件是否存在

```bash
# 1. 登录服务器
ssh root@服务器IP

# 2. 检查图片是否存在
ls -lh /var/www/hexo-blog/source/img/photo.jpg

# 3. 查看目录下所有文件
ls -lh /var/www/hexo-blog/source/img/
```

#### 方法 2：检查生成后的文件

```bash
# 生成网站后，检查 public 目录
ls -lh /var/www/hexo-blog/public/img/photo.jpg

# 如果文件存在，说明路径正确
```

#### 方法 3：浏览器直接访问

```
访问：http://你的域名/img/photo.jpg

# 如果能看到图片，说明路径正确
# 如果 404，说明文件不存在或路径错误
```

### 4.6 路径命名建议

#### 推荐的文件命名规范

```bash
# 1. 使用日期前缀（方便管理）
2024-01-15-baby-smile.jpg
2024-02-20-first-step.mp4

# 2. 使用描述性名称
baby-first-birthday.jpg
learning-to-walk.mp4

# 3. 避免中文文件名（可能导致编码问题）
❌ 不推荐：宝贝的笑容.jpg
✅ 推荐：baby-smile.jpg

# 4. 避免空格和特殊字符

```

## 九、视频播放器使用

### 9.1 视频文件准备

上传视频到 `source/videos/` 目录，并准备封面图到 `source/img/` 目录

### 9.2 基本用法

**仅视频（无封面）**：
```markdown
{% dplayer "url=/videos/baby-video.mp4" %}
```

**带封面图（推荐）**：
```markdown
{% dplayer "url=/videos/baby-video.mp4" "pic=/img/video-cover.jpg" %}
```

### 9.3 参数说明

| 参数 | 作用 | 必需 | 示例 |
|------|------|------|------|
| url | 视频路径 | ✅ | url=/videos/baby.mp4 |
| pic | 封面图（播放前显示） | ❌ | pic=/img/cover.jpg |
| autoplay | 自动播放 | ❌ | autoplay=true |
| loop | 循环播放 | ❌ | loop=true |
| volume | 音量 0-1 | ❌ | volume=0.7 |

**pic 参数详解**：
- 作用：设置视频播放前显示的封面图片
- 不设置时：显示黑屏或视频第一帧
- 推荐使用：提升视觉效果，避免黑屏
- 图片尺寸：建议 16:9 比例（如 1920x1080）

### 9.4 完整示例

```markdown
## 宝贝学走路

{% dplayer "url=/videos/first-step.mp4" "pic=/img/first-step-cover.jpg" "autoplay=false" "loop=false" "volume=0.7" %}

今天宝贝迈出了第一步！
```
